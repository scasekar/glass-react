4. Create hand-rolled control components in `demo/controls/`:

**SliderControl.tsx**: A labeled range input showing current value. Props: label, value, min, max, step, onChange. Displays value formatted to 2 decimal places. Inline styles only (no CSS files).

**ColorControl.tsx**: Three range inputs (R, G, B) each 0-1 with step 0.01, plus a color preview swatch. Props: label, value (GlassColor tuple), onChange. The swatch shows the current color as a small square next to the label.

**SelectControl.tsx**: A labeled `<select>` dropdown. Props: label, value, options (array of {value, label}), onChange. Styled consistently with slider controls.

**ControlPanel.tsx**: A collapsible sidebar/panel that contains all controls. Uses useState for open/closed state. Contains sections:
- **Blur & Opacity**: blur (0-1, step 0.01, default 0.5), opacity (0-1, step 0.01, default 0.05)
- **Geometry**: cornerRadius (0-50, step 1, default 24)
- **Refraction**: refraction mode (standard/prominent dropdown), refraction strength (not directly exposed but implicit in mode)
- **Visual Effects**: aberration (0-10, step 0.5, default 3), specular (0-1, step 0.01, default 0.2), rim (0-1, step 0.01, default 0.15)
- **Color**: tint color (RGB control, default [1,1,1])
- **Animation**: morphSpeed (0-20, step 1, default 8)

The ControlPanel takes the current values as props and calls onChange callbacks to update them. It should be positioned as a fixed sidebar on the right side, ~280px wide, with a toggle button visible when collapsed. Style it with a semi-transparent dark background so it doesn't obstruct the glass demo but is clearly readable.

5. Create `demo/App.tsx`:
The demo App uses `useState` to hold all glass parameters (GlassStyleProps). It renders:
- `<GlassProvider>` wrapping everything
- A centered layout with all three glass components:
  - `<GlassPanel>` with some content text and the current parameter values applied
  - Two `<GlassButton>` instances (one standard, one prominent mode)
  - `<GlassCard>` with content
- The `<ControlPanel>` component with all parameters wired to state
- A title/header area showing "LiquidGlass React" and a brief description
- All glass components receive their props from the shared state (except buttons which may override refractionMode)

Import components from the source directly (`../src/components/...`) for dev mode — this gives HMR. The demo is a development tool, not a consumer simulation.

Keep all existing scripts. These are additive.

  <action>
Create `API.md` at the project root with comprehensive documentation for the public API. Structure:

1. **Quick Start** — 10-line example showing GlassProvider + GlassPanel usage (the consumer example from research). Include the npm install command.

2. **Components**:

   **GlassProvider** — Required wrapper component. Document: must wrap all glass components, creates the WebGPU canvas automatically, only ONE per page (hardcoded canvas ID constraint). Props: children only. Mention it handles engine lifecycle, resize, and accessibility preferences automatically.

   **GlassPanel** — Renders as a `<div>`. Document all GlassStyleProps inherited plus className, style, ref, children. Show a usage example.

   **GlassButton** — Renders as a `<button>`. Document all GlassStyleProps plus onClick, disabled, type, className, style, ref, children. Note the built-in hover/active morph states (specular 1.8x, rim 2x, aberration 1.5x on hover; blur 0.3x on press). Show a usage example.

   **GlassCard** — Renders as an `<article>`. Document all GlassStyleProps plus className, style, ref, children. Show a usage example.

3. **Shared Props (GlassStyleProps)** — Table format:
   | Prop | Type | Default | Range | Description |
   For each: blur, opacity, cornerRadius, tint, refraction, aberration, specular, rim, refractionMode, morphSpeed.

4. **Types** — List all exported types with their definitions:
   - GlassStyleProps, GlassPanelProps, GlassButtonProps, GlassCardProps, GlassColor, AccessibilityPreferences
   - GlassContextValue, GlassRegionHandle (advanced)

5. **Hooks** (Advanced):
   - `useGlassEngine()` — returns GlassContextValue. Must be used within GlassProvider. For advanced use cases where you need direct access to the engine context.

6. **Accessibility** — Document automatic behaviors:
   - prefers-reduced-motion: background animation freezes
   - prefers-reduced-transparency: components render near-opaque
   - prefers-color-scheme: tint adapts to dark/light mode (when no explicit tint prop provided)
   - WCAG 2.1 AA contrast: text automatically gets contrast-safe styling

7. **Requirements** — WebGPU-capable browser (Chrome 113+, Edge 113+), React 18 or 19.

8. **Limitations** — Single GlassProvider per page, no SSR support, no non-WebGPU fallback.

Write in clear, technical prose. No marketing fluff. Include TypeScript code examples for each component. All prop descriptions should match the JSDoc in types.ts.
