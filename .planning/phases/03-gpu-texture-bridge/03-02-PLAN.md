  <files>engine/src/main.cpp, src/wasm/loader.ts, src/App.tsx</files>
  <action>
1. Update `engine/src/main.cpp`:
   - Add `destroyEngine()` free function above the EMSCRIPTEN_BINDINGS block:
     ```cpp
     void destroyEngine() {
         if (g_engine) {
             delete g_engine;
             g_engine = nullptr;
         }
     }
     ```
   - Add to EMSCRIPTEN_BINDINGS block:
     ```cpp
     emscripten::function("destroyEngine", &destroyEngine);
     ```
   - The BackgroundEngine destructor is `= default`, which is fine -- wgpu::Device, wgpu::Texture, etc. are C++ RAII wrappers that release GPU resources when destroyed.

2. Update `src/wasm/loader.ts`:
   - Add `destroyEngine()` to the EngineModule interface:
     ```typescript
     interface EngineModule {
       getEngine(): { resize(w: number, h: number): void } | null;
       destroyEngine(): void;
     }
     ```
   - Note: getEngine() return type should include `| null` since after destroyEngine() it returns nullptr.

3. Update `src/App.tsx`:
   - Store the module reference in a ref so the cleanup function can access it:
     ```typescript
     const moduleRef = useRef<EngineModule | null>(null);
     ```
     (EngineModule must be imported or the type inlined)
   - In the initEngine().then() callback, store: `moduleRef.current = module;`
   - In the useEffect cleanup function, add engine destruction:
     ```typescript
     return () => {
       cancelled = true;
       observerRef.current?.disconnect();
       if (moduleRef.current) {
         moduleRef.current.destroyEngine();
         moduleRef.current = null;
       }
     };
     ```
   - Export the EngineModule interface from loader.ts (add `export` keyword) so App.tsx can import it for the ref type. OR inline the type in App.tsx to avoid circular concerns -- use your judgment.

4. Build WASM: `cd engine/build-web && emcmake cmake .. && make -j$(nproc)`
