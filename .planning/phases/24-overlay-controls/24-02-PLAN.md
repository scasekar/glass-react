    **Step 1: Write failing tests in `src/components/__tests__/GlassSheet.test.tsx`.**

    Use `@vitest-environment jsdom` directive. Mock `useGlassEngine` and `useGlassRegion` following the established pattern. Mock `motion/react`: `AnimatePresence` as passthrough, `motion.div` as a regular div that forwards all props (including data-testid). Write tests for the 6 behaviors above.

    **Step 2: Implement `src/components/controls/GlassSheet.tsx`.**

    Follow Radix Dialog + forceMount + AnimatePresence pattern:
    - Use `Dialog.Root` with controlled `open`/`onOpenChange`
    - Gate portal content with `AnimatePresence > {open && <Dialog.Portal forceMount>...}`
    - `Dialog.Overlay asChild forceMount` wrapping `motion.div` with fade animation, fixed inset 0, background `rgba(0,0,0,0.4)`, zIndex 9998, onClick calls `onOpenChange(false)`
    - `Dialog.Content asChild forceMount` wrapping `motion.div` with slide-up animation (y: '100%' to y: 0), positioned fixed bottom 0 left 0 right 0, zIndex 9999
    - Spring transition: `{ type: 'spring', damping: 30, stiffness: 300 }`
    - Height: `height === 'full' ? '100vh' : '50vh'` on the content container

    Inside the motion.div content container:
    - `<GlassPanel cornerRadius={APPLE_RADII.xl}` with `borderBottomLeftRadius: 0, borderBottomRightRadius: 0` via style, full height>
    - **Drag handle (CRITICAL — Pitfall 5):** If `showDragHandle !== false`, render a non-scrollable drag handle area at top:
      ```tsx
      <motion.div
        drag="y"
        dragConstraints={{ top: 0 }}
        dragElastic={{ top: 0, bottom: 0.3 }}
        onDragEnd={(_, info) => {
          if (info.offset.y > 100 || info.velocity.y > 500) {
            onOpenChange(false);
          }
        }}
        style={{ padding: APPLE_SPACING.md, cursor: 'grab', touchAction: 'none' }}
        data-testid="glass-sheet-drag-handle"
      >
        <div style={{
          width: 36, height: 5, borderRadius: 2.5,
          background: 'rgba(255,255,255,0.3)', margin: '0 auto'
        }} />
      </motion.div>
      ```
    - The drag handle is separate from content — content area below is a scrollable div with `overflow: 'auto'`, `flex: 1`. This avoids Pitfall 5 (drag competes with scroll).
    - Optional `<Dialog.Title>` if `title` prop provided. If no title, add `<Dialog.Title className="sr-only">Sheet</Dialog.Title>` to satisfy Radix.
    - `<Dialog.Description className="sr-only">` with "Modal sheet" text to suppress Radix warning.
    - `{children}` inside the scrollable area.
  </action>
