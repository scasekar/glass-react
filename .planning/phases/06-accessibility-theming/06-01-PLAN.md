    - `void setPaused(bool paused);` -- public method
    - `void setReducedTransparency(bool enabled);` -- public method
    - `bool paused_ = false;` -- private member variable
    - `bool reducedTransparency_ = false;` -- private member variable

    **background_engine.cpp:** Implement the two methods:
    - `setPaused(bool paused)`: Sets `paused_` flag. In the existing `update(float deltaTime)` method, add an early return `if (paused_) return;` BEFORE the deltaTime cap and `currentTime +=` line. This freezes the `time` uniform at its current value. The render loop continues running (do NOT use emscripten_pause_main_loop) so glass regions still track DOM positions.
    - `setReducedTransparency(bool enabled)`: Sets `reducedTransparency_` flag. This flag is stored but not acted upon in C++ -- the React side will override glass region parameters (opacity, tint, refraction) when this is true. The C++ side just stores the state for potential future use.

    **IMPORTANT for update():** The early return must be the FIRST thing in update(), before the deltaTime > 0.1f cap. The existing code is:
    ```cpp
    void BackgroundEngine::update(float deltaTime) {
        if (deltaTime > 0.1f) { deltaTime = 0.1f; }
        currentTime += deltaTime;
    }
    ```
    Change to:
    ```cpp
    void BackgroundEngine::update(float deltaTime) {
        if (paused_) return;
        if (deltaTime > 0.1f) { deltaTime = 0.1f; }
        currentTime += deltaTime;
    }
    ```

    **main.cpp:** Add Embind bindings for both new methods in the existing `emscripten::class_<BackgroundEngine>` block:
    - `.function("setPaused", &BackgroundEngine::setPaused)`
    - `.function("setReducedTransparency", &BackgroundEngine::setReducedTransparency)`

    **src/wasm/loader.ts:** Add the two new methods to the `EngineModule.getEngine()` return type:
    - `setPaused(paused: boolean): void;`
    - `setReducedTransparency(enabled: boolean): void;`

    **src/components/types.ts:** Add `AccessibilityPreferences` interface at the top of the file (before GlassColor):
    ```typescript
    /** OS-level accessibility and theme preferences detected via matchMedia */
    export interface AccessibilityPreferences {
      /** true when OS has prefers-reduced-motion: reduce */
      reducedMotion: boolean;
      /** true when OS has prefers-reduced-transparency: reduce (progressive enhancement, ~72% browser support) */
      reducedTransparency: boolean;
      /** true when OS has prefers-color-scheme: dark */
      darkMode: boolean;
    }
    ```

    **src/hooks/useAccessibilityPreferences.ts:** Create new file implementing the hook using `useSyncExternalStore` (NOT useEffect+useState -- concurrent-safe per research). Follow the pattern from 06-RESEARCH.md exactly:
    - Create a `createMediaQueryStore(query: string)` factory that returns `{subscribe, getSnapshot, getServerSnapshot}`.
    - `subscribe` uses `addEventListener('change', callback)` and returns cleanup that calls `removeEventListener('change', callback)`. Create a NEW `window.matchMedia(query)` inside subscribe (not cached) so each subscribe/unsubscribe manages its own listener.
    - `getSnapshot` returns `window.matchMedia(query).matches`.
    - `getServerSnapshot` returns `false` (SSR fallback, no window).
    - Create THREE module-level store instances:
      - `reducedMotionStore = createMediaQueryStore('(prefers-reduced-motion: reduce)')`
      - `reducedTransparencyStore = createMediaQueryStore('(prefers-reduced-transparency: reduce)')`
      - `darkModeStore = createMediaQueryStore('(prefers-color-scheme: dark)')`
    - Export `useAccessibilityPreferences()` that calls `useSyncExternalStore` three times (once per store) and returns `AccessibilityPreferences`.
    - Import `AccessibilityPreferences` type from `../components/types`.

    **src/context/GlassContext.ts:** Add `preferences` to `GlassContextValue`:
    - Import `AccessibilityPreferences` from `../components/types`.
    - Add `preferences: AccessibilityPreferences;` to the `GlassContextValue` interface.

    **src/components/GlassProvider.tsx:** Wire everything together:
    1. Import `useAccessibilityPreferences` from `../hooks/useAccessibilityPreferences`.
    2. Call `const prefs = useAccessibilityPreferences();` inside the component.
    3. Add a useEffect that syncs `prefs.reducedMotion` to the engine:
       ```typescript
       useEffect(() => {
         const engine = moduleRef.current?.getEngine();
         if (!engine) return;
         engine.setPaused(prefs.reducedMotion);
       }, [prefs.reducedMotion, ready]);
       ```
    4. Add the `preferences` field to the context value `useMemo`:
       ```typescript
       const contextValue = useMemo(() => ({
         registerRegion,
         unregisterRegion,
         ready,
         preferences: prefs,
       }), [registerRegion, unregisterRegion, ready, prefs]);
       ```
    Note: Do NOT call `engine.setReducedTransparency()` from GlassProvider. The reduced-transparency adaptation is handled entirely in React (Plan 02 will override glass params in useGlassRegion). The engine method exists for future use but is not called here.
  </action>
