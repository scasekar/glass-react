  <action>
    **src/utils/contrast.ts:** Create new file with WCAG 2.1 contrast ratio utilities. Follow the W3C spec formulas from 06-RESEARCH.md exactly:
    - `sRGBtoLinear(c: number): number` -- internal helper, converts sRGB component (0-1) to linear light
    - `relativeLuminance(r: number, g: number, b: number): number` -- exported, computes relative luminance per W3C formula (0.2126 * R_lin + 0.7152 * G_lin + 0.0722 * B_lin)
    - `contrastRatio(l1: number, l2: number): number` -- exported, computes (lighter + 0.05) / (darker + 0.05)
    - `meetsWCAG_AA(textColor: [number, number, number], bgColor: [number, number, number]): boolean` -- exported, returns true if contrast ratio >= 4.5

    **src/hooks/useGlassRegion.ts:** Modify to read accessibility preferences from context and compute effective glass parameters:

    1. Import `AccessibilityPreferences` from `../components/types` and `GlassColor` from `../components/types`.

    2. Add dark/light mode default constants at module level (NOT inside the hook):
       ```typescript
       const DARK_DEFAULTS = {
         tint: [0.7, 0.75, 0.85] as GlassColor,  // cool blue-white
         opacity: 0.08,
       };
       const LIGHT_DEFAULTS = {
         tint: [0.15, 0.15, 0.2] as GlassColor,  // dark charcoal
         opacity: 0.25,
       };
       ```

    3. Add reduced-transparency override constants:
       ```typescript
       const REDUCED_TRANSPARENCY_DARK = {
         tint: [0.2, 0.2, 0.22] as GlassColor,
         opacity: 0.92,
       };
       const REDUCED_TRANSPARENCY_LIGHT = {
         tint: [0.92, 0.92, 0.94] as GlassColor,
         opacity: 0.90,
       };
       ```

    4. Modify the existing prop-sync useEffect to use effective parameters instead of raw props. The current code directly sends props to the engine. Change it to compute effective values first:

       **When `prefs.reducedTransparency` is true:**
       - `blur`: 0 (no blur, solid surface)
       - `opacity`: from REDUCED_TRANSPARENCY_DARK/LIGHT based on darkMode
       - `tint`: from REDUCED_TRANSPARENCY_DARK/LIGHT based on darkMode (ONLY if user did NOT provide explicit tint -- if `props.tint` is defined, use it)
       - `refraction`: 0 (no refraction when opaque)
       - `cornerRadius`: from props or default

       **When `prefs.reducedTransparency` is false (normal mode):**
       - `blur`: `props.blur ?? 0.5`
       - `opacity`: `props.opacity ?? defaults.opacity` (where defaults = DARK_DEFAULTS or LIGHT_DEFAULTS based on darkMode)
       - `tint`: `props.tint ?? defaults.tint` (ONLY apply dark/light default when user has NOT provided explicit tint)
       - `refraction`: `props.refraction ?? 0.15`
       - `cornerRadius`: `props.cornerRadius ?? 24`

    5. Access preferences from context: `const { preferences: prefs } = ctx;` (ctx is already available from `useGlassEngine()`).

    6. Update the useEffect dependency array to include `prefs.reducedTransparency` and `prefs.darkMode` so params recalculate when OS preferences change.

    **CRITICAL:** When user provides explicit `props.tint`, that value MUST be used in ALL modes (normal, dark, light, reduced-transparency). Dark/light defaults and reduced-transparency tints are ONLY applied when `props.tint` is `undefined`.
