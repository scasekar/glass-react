The shader must contain:

- Vertex shader `vs_main` using fullscreen triangle pattern (3 hardcoded vertices from `vertex_index`, positions covering clip space [-1,1], UVs [0,1]). Use the exact pattern from the research: positions (-1,-1), (3,-1), (-1,3).
- `simplexNoise2(v: vec2f) -> f32` function ported from munrocket's gist (mod289_2, mod289_3, permute3, simplexNoise2 -- use the exact code from 02-RESEARCH.md Code Examples section)
- `fbm(st: vec2f, time: f32) -> f32` function with 6 octaves, lacunarity 2.0, persistence 0.5, time offset `time * 0.15`
- Fragment shader `fs_main` that: normalizes UVs by aspect ratio (`uv * resolution / min(resolution.x, resolution.y)`), calls `fbm(st * 3.0, uniforms.time)`, maps result to a blue/teal color ramp: `vec3f(n * 0.15, n * 0.3 + 0.05, n * 0.6 + 0.1)` (dark blue to teal gradient). Multiply noise output `* 0.5 + 0.5` to remap from [-1,1] to [0,1] before color mapping.

**engine/src/background_engine.h** -- BackgroundEngine class declaration:
```cpp
#pragma once
#include <webgpu/webgpu_cpp.h>
#include <cstdint>

struct Uniforms {
    float time;
    float _pad1;
    float resolutionX;
    float resolutionY;
};

class BackgroundEngine {
public:
    BackgroundEngine();
    ~BackgroundEngine();
    void init(wgpu::Device dev, wgpu::Surface surf, wgpu::TextureFormat fmt, uint32_t w, uint32_t h);
    void update(float deltaTime);
    void render();
    void resize(uint32_t newWidth, uint32_t newHeight);
private:
    void createPipeline();
    void createUniforms();
    wgpu::Device device;
    wgpu::Surface surface;
    wgpu::TextureFormat surfaceFormat;
    wgpu::ShaderModule shaderModule;
    wgpu::RenderPipeline pipeline;
    wgpu::BindGroupLayout bindGroupLayout;
    wgpu::BindGroup bindGroup;
    wgpu::Buffer uniformBuffer;
    uint32_t width = 0;
    uint32_t height = 0;
    float currentTime = 0.0f;
};
```

**engine/src/background_engine.cpp** -- Full implementation:
- `init()`: Store device/surface/format/dimensions, call `createPipeline()` then `createUniforms()`
- `createPipeline()`: Create shader module using `wgpu::ShaderSourceWGSL` (NOT old ShaderModuleWGSLDescriptor -- this is critical). Create explicit `BindGroupLayout` with one entry (binding 0, vertex+fragment visibility, uniform buffer type, minBindingSize = sizeof(Uniforms)). Create `PipelineLayout` from the bind group layout. Create `RenderPipeline` with: vertex stage (vs_main), fragment stage (fs_main), color target matching surfaceFormat, TriangleList topology, multisample count 1 mask ~0u.
- `createUniforms()`: Create uniform buffer (size sizeof(Uniforms), usage CopyDst|Uniform). Create bind group with the buffer at binding 0.
- `update(float deltaTime)`: Accumulate `currentTime += deltaTime`. Cap deltaTime to 0.1f before accumulating to prevent jumps on tab switch.
- `render()`: Write uniforms to buffer via `queue.WriteBuffer()`. Get surface texture. Create render pass with clear color {0,0,0,1}. Set pipeline, set bind group 0, draw 3 vertices. Submit.
- `resize()`: Guard against 0 dimensions. Update width/height. Reconfigure surface with new dimensions (`surface.Configure()`). No offscreen texture needed for Phase 2 (per research recommendation -- render directly to surface, defer offscreen texture to Phase 3).
- Destructor: Default (wgpu:: wrappers handle cleanup via RAII).

IMPORTANT API notes from Phase 1 learnings:
- Use `wgpu::ShaderSourceWGSL` not `wgpu::ShaderModuleWGSLDescriptor`
- Use PascalCase methods: `CreateShaderModule`, `CreateBuffer`, `CreateRenderPipeline`
- Use designated initializer syntax where possible (matches Phase 1 style)
- `wgpu::ShaderSourceWGSL` goes on `shaderDesc.nextInChain`

**engine/src/main.cpp** -- Rewrite to use BackgroundEngine class:
- Remove all global WebGPU state (instance, adapter, device, surface, format, initComplete)
- Remove the old Render() function
- Remove InitGraphics() -- the BackgroundEngine handles pipeline setup
- Keep the same async device acquisition pattern: `wgpu::CreateInstance()` -> `RequestAdapter(AllowSpontaneous)` -> `RequestDevice(AllowSpontaneous)`
- Keep `wgpu::EmscriptenSurfaceSourceCanvasHTMLSelector` for surface creation with `#gpu-canvas`
- Keep `emscripten_set_main_loop(MainLoop, 0, false)` in main()
- Add a global `BackgroundEngine* g_engine = nullptr` pointer
- In `OnDeviceAcquired`: create surface, get capabilities/format, create `g_engine = new BackgroundEngine()`, call `g_engine->init(device, surface, format, 512, 512)`. Use 512x512 as initial size (JS will resize immediately via ResizeObserver in Plan 02).
- In `MainLoop`: calculate delta time using `emscripten_get_now()` (returns ms, convert to seconds). Use a `static double lastTime = 0.0`. On first frame (lastTime == 0), set lastTime = now and return (skip first frame to avoid huge dt). Cap dt to 0.1f. Call `g_engine->update(dt)` then `g_engine->render()`.
- Add Embind bindings at the bottom of main.cpp:
```cpp
#include <emscripten/bind.h>
EMSCRIPTEN_BINDINGS(background_engine) {
    emscripten::class_<BackgroundEngine>("BackgroundEngine")
        .function("resize", &BackgroundEngine::resize);
}
```
Note: Only expose `resize()` via Embind for now. The `init()` is called internally during device acquisition. The `update()/render()` are called by the main loop. JS only needs `resize()` to handle canvas resize events. Do NOT expose a constructor via Embind -- the engine is created internally and managed by the C++ main loop. Instead, expose a free function `getEngine()` that returns a pointer:
```cpp
BackgroundEngine* getEngine() { return g_engine; }
EMSCRIPTEN_BINDINGS(background_engine) {
    emscripten::function("getEngine", &getEngine, emscripten::allow_raw_pointers());
    emscripten::class_<BackgroundEngine>("BackgroundEngine")
        .function("resize", &BackgroundEngine::resize);
}
```
