**src/wasm/loader.ts** -- Return the Emscripten Module object so App.tsx can access Embind bindings:
```typescript
interface EngineModule {
  getEngine(): { resize(w: number, h: number): void };
}

export async function initEngine(): Promise<EngineModule> {
  if (!navigator.gpu) {
    throw new Error('WebGPU is not supported in this browser. Use Chrome 113+ or Edge 113+.');
  }

  // @ts-expect-error -- Emscripten generated module, no type declarations
  const createEngineModule = (await import('../../engine/build-web/engine.js')).default;
  const module = await createEngineModule();
  return module as EngineModule;
}
```
The key change: return `module` instead of void. The module object has `getEngine()` from the Embind free function.

**src/App.tsx** -- Wire ResizeObserver to engine.resize():
- `initEngine()` now returns the module. Store it in a ref (`useRef`).
- After engine loads successfully, get the engine handle: `const engine = module.getEngine()`
- Set up a `ResizeObserver` on the canvas element (`document.getElementById('gpu-canvas')`).
- In the ResizeObserver callback (use the pattern from 02-RESEARCH.md JS ResizeObserver section):
  - Get pixel dimensions using `devicePixelContentBoxSize` with fallback to `contentBoxSize * devicePixelRatio`
  - Clamp to range [1, 4096]
  - Only call resize if dimensions actually changed: `if (canvas.width !== w || canvas.height !== h)`
  - Update `canvas.width = w; canvas.height = h;` (buffer size)
  - Call `engine.resize(w, h)`
- Store the observer in a ref. Clean up in the useEffect return: `observer.disconnect()`.
- Try `observer.observe(canvas, { box: 'device-pixel-content-box' })` with fallback to `{ box: 'content-box' }`.
- Remove the status display text (no more "Engine running" paragraph -- the noise background IS the visual confirmation). Or move it to a small subtle indicator in the corner if desired, but the main visual is the canvas itself.
- Keep error display for WebGPU not supported / init failure.

IMPORTANT: The ResizeObserver fires immediately on observe(), which triggers the first resize from 512x512 to the actual viewport size. This is the expected behavior -- the C++ engine starts at 512x512 and the ResizeObserver corrects it within the first frame.

Do NOT call `engine.delete()` in cleanup -- the engine is owned by the C++ global pointer and managed by the main loop, not by JS Embind object lifecycle.
