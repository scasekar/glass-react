  <action>
Create `pipeline/lib/scorer.ts` that provides a persistent Playwright browser for fast repeated captures and a scoring function that combines capture + normalize + compare into a single call.

**CaptureContext interface and lifecycle:**
- `CaptureContext` holds `browser`, `context`, and `page` from Playwright
- `createCaptureContext(mode: 'light' | 'dark', viewport: { width: number; height: number })` launches Chrome with `channel: 'chrome'` and GPU flags `['--enable-gpu', '--use-gl=egl']`, creates a browser context with `deviceScaleFactor: 1` and the appropriate `colorScheme`, and opens a page. Returns `CaptureContext`.
- `closeCaptureContext(ctx: CaptureContext)` closes the browser gracefully.

**captureWithParams helper (not exported):**
- Takes `CaptureContext`, `baseUrl: string`, `params: GlassParams`
- Builds URL: `${baseUrl}/?${paramsToQueryString(params)}` where `paramsToQueryString` serializes all GlassParams (including `capture=true` flag). Implement the serialization inline (the existing `presetToQueryString` in config.ts is module-private and takes a preset name, not a GlassParams object). Handle `tint` as `JSON.stringify(value)`, all others as `String(value)`.
- Navigates via `page.goto(url)`
- Waits for `#gpu-canvas` visible, then `waitForFunction` for canvas width > 0 and height > 0
- Settle time: 2000ms (warm browser, per research recommendation; shorter than captureWeb's 3s cold start)
- Screenshots the canvas element, returns `Buffer`

**createScorer factory:**
- Signature: `createScorer(ctx: CaptureContext, baseUrl: string, iosReferencePath: string, mode: 'light' | 'dark'): (params: GlassParams) => Promise<number>`
- Uses temp files in os.tmpdir() for web raw, web normalized, and diff images (use `path.join(os.tmpdir(), 'tune_web_raw.png')` etc.)
- Flow: capture screenshot with params -> write to temp -> normalize(tempRaw, tempNorm) -> compare(tempNorm, iosReferencePath, tempDiff, CONFIG.roiMask) -> return result.percentage
- Import `normalize` from `./normalize.js`, `compare` from `./compare.js`, `CONFIG` from `./config.js`

**Imports:** `playwright` (chromium), `fs`, `os`, `path`, types from `./config.js` and `../../demo/controls/presets.js`

**Types to define:**
```typescript
interface TuningParam {
  key: string;  // GlassParams key or 'tint_r' | 'tint_g' | 'tint_b'
  min: number;
  max: number;
  step: number;
  minStep: number;
}

interface TuningConfig {
  params: TuningParam[];
  maxCycles: number;           // Default 15
  convergenceThreshold: number; // Default 0.05 (stop if improvement < 0.05% per cycle)
}

interface IterationEntry {
  cycle: number;
  paramIndex: number;
  param: string;
  direction: 'plus' | 'minus' | 'none';
  step: number;
  oldValue: number;
  newValue: number;
  oldScore: number;
  newScore: number;
  delta: number;       // newScore - oldScore (negative = improved)
  bestScore: number;
  timestamp: string;   // ISO 8601
}

interface TuningResult {
  mode: 'light' | 'dark';
  startingPreset: string;
  initialScore: number;
  finalScore: number;
  improvement: number;
  cycles: number;
  totalEvaluations: number;
  elapsed: string;     // HH:MM:SS
  bestParams: GlassParams;
  log: IterationEntry[];
}
```

**Helper functions (not exported):**
- `getParamValue(params: GlassParams, key: string): number` -- for tint_r/g/b, reads from `params.tint[0/1/2]`; for all other keys, reads `params[key] as number`
- `setParamValue(params: GlassParams, key: string, value: number): GlassParams` -- returns new GlassParams with the value set. For tint_r/g/b, creates a new tint array with the component updated. For other keys, uses spread.

**coordinateDescent function:**
- Signature: `async function coordinateDescent(initialParams: GlassParams, config: TuningConfig, mode: 'light' | 'dark', presetName: string, score: (params: GlassParams) => Promise<number>, onIteration?: (entry: IterationEntry) => void): Promise<TuningResult>`
- Takes an optional `onIteration` callback for real-time logging
- Algorithm:
  1. Score initial params to get baseline
  2. Copy step sizes from config into a mutable `steps` record
  3. For each cycle (0 to maxCycles-1):
     a. Record cycleStartScore
     b. For each param in config.params:
        - Get current value, compute +step (clamped to max) and -step (clamped to min)
        - Score plusParams, score minusParams
        - If plusScore < bestScore and plusScore <= minusScore: accept plus
        - Else if minusScore < bestScore: accept minus
        - Else: direction = 'none'
        - Push IterationEntry to log, call onIteration if provided
     c. If no param improved in this cycle: halve all steps; if all below minStep, break
     d. If |cycleStartScore - bestScore| < convergenceThreshold: break
  4. Compute elapsed time as HH:MM:SS
  5. Return TuningResult with all fields populated

**Count totalEvaluations:** Increment a counter for each `score()` call (2 per param per cycle, plus 1 for initial baseline).

**Export:** `coordinateDescent`, `TUNABLE_PARAMS`, and all type interfaces (`TuningConfig`, `TuningParam`, `IterationEntry`, `TuningResult`).
