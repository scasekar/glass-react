  <name>Task 1: Convert BackgroundEngine to multi-region rendering with dynamic uniform buffer offsets</name>
  <files>engine/src/background_engine.h, engine/src/background_engine.cpp</files>
  <action>
**In background_engine.h:**

1. Add constant: `static constexpr uint32_t MAX_GLASS_REGIONS = 16;`

2. Add GlassRegion struct:
```cpp
struct GlassRegion {
    GlassUniforms uniforms{};
    bool active = false;
};
```

3. Replace the single-region members with multi-region members:
   - Remove: `GlassUniforms glassUniforms{};`
   - Add: `GlassRegion regions[MAX_GLASS_REGIONS]{};`
   - Add: `uint32_t uniformStride = 0;` (aligned stride between regions in the buffer)

4. Replace old single-region public API (`setGlassRect`, `setGlassParams`, `setGlassTint`) with multi-region API:
   - `int addGlassRegion();` -- finds first inactive slot, marks active, returns index (0-15) or -1 if full
   - `void removeGlassRegion(int id);` -- marks slot inactive (bounds-checks 0-15)
   - `void setRegionRect(int id, float x, float y, float w, float h);`
   - `void setRegionParams(int id, float cornerRadius, float blur, float opacity, float refraction);`
   - `void setRegionTint(int id, float r, float g, float b);`

**In background_engine.cpp:**

1. In `init()`:
   - Query device limits: `wgpu::SupportedLimits limits; device.GetLimits(&limits);`
   - Compute aligned stride: `uniformStride = ceilToNextMultiple(static_cast<uint32_t>(sizeof(GlassUniforms)), static_cast<uint32_t>(limits.limits.minUniformBufferOffsetAlignment));`
   - Write a small `ceilToNextMultiple` helper (anonymous namespace or inline): `return (value + alignment - 1) / alignment * alignment;`
   - Remove the old single-region default initialization code (the lines setting glassUniforms.rectX etc.)

2. In `createGlassPipeline()`:
   - Change the uniform buffer layout entry: set `entries[2].buffer.hasDynamicOffset = true;`
   - Change `entries[2].buffer.minBindingSize` to remain `sizeof(GlassUniforms)` (this is the per-draw binding size, not total buffer size)
   - Resize the glass uniform buffer: `bufDesc.size = uniformStride * MAX_GLASS_REGIONS;`

3. In `createGlassBindGroup()`:
   - Change entries[2].size from `sizeof(GlassUniforms)` to `sizeof(GlassUniforms)` (the per-draw binding window -- stays the same)
   - Bind group creation stays the same -- the dynamic offset is set per draw call, not in the bind group

4. In `render()` -- glass pass:
   - Replace the single `device.GetQueue().WriteBuffer(glassUniformBuffer, 0, ...)` and single `pass.Draw(3)` with a loop:
   ```cpp
   for (uint32_t i = 0; i < MAX_GLASS_REGIONS; i++) {
       if (!regions[i].active) continue;
       regions[i].uniforms.resolutionX = static_cast<float>(width);
       regions[i].uniforms.resolutionY = static_cast<float>(height);
       device.GetQueue().WriteBuffer(glassUniformBuffer, i * uniformStride,
                                     &regions[i].uniforms, sizeof(GlassUniforms));
       uint32_t dynamicOffset = i * uniformStride;
       pass.SetBindGroup(0, glassBindGroup, 1, &dynamicOffset);
       pass.Draw(3);
   }
   ```
   - If no regions are active, still render a passthrough (draw once with a zeroed-out region that has opacity=0) OR skip the glass pass entirely and render the offscreen texture directly. Simplest: just skip the draw calls, and change the glass pass loadOp to Load (so the surface keeps the noise texture). Actually, even simpler: render with the background directly. The current glass shader outputs passthrough outside the glass rect. With no active regions, the surface would be black. **Solution:** Before the glass pass loop, do a simple full-screen blit of the offscreen texture to the surface (or use loadOp with the offscreen texture). The cleanest approach: if there are no active regions, draw once with the full pipeline using a region with rectW=1, rectH=1, opacity=0, blur=0, refraction=0 -- the shader will pass through the background. **Even better:** draw one pass with zero active regions should produce the passthrough background. Let's verify: the glass shader does `mix(bgColor, glassColor, mask)`. If rectW=0, rectH=0, the SDF will be negative everywhere, mask=0, output=bgColor. So: always draw at least one pass with a dummy region (rectW=0) if no active regions exist, OR rely on the fact that any active region's passthrough covers non-glass areas. **Simplest correct approach:** always iterate all regions; inactive regions are skipped; if zero regions are active, skip the draw calls entirely and the loadOp=Clear leaves black. But we need the background! So: change the glass pass to first blit the background, then overdraw glass regions. Wait -- the current shader already outputs passthrough outside the glass rect. So each draw call renders the full background PLUS glass in its rect. With multiple draw calls, the last one "wins" for background areas. This is fine because they all output the same background outside their rect. **The render loop is correct as-is for multiple regions.** If zero regions active, we need at least one draw. Add a fallback: if no regions were drawn, do one draw with a dummy region (all zeros -- rectW=0, rectH=0 means mask=0 everywhere, pure passthrough).

5. Implement the new multi-region API methods:
   - `addGlassRegion()`: linear scan `regions[]` for first `!active`, mark active, return index. Set default uniforms on the new region (cornerRadius=24, blur=0.5, opacity=0.05, refraction=0.15, tint=white, aberration=3.0). Return -1 if all 16 full.
   - `removeGlassRegion(int id)`: bounds-check `0 <= id < MAX_GLASS_REGIONS`, set `regions[id].active = false`.
   - `setRegionRect(int id, ...)`: bounds-check, set `regions[id].uniforms.rectX/Y/W/H`.
   - `setRegionParams(int id, ...)`: bounds-check, set `regions[id].uniforms.cornerRadius/blurIntensity/opacity/refractionStrength`.
   - `setRegionTint(int id, ...)`: bounds-check, set `regions[id].uniforms.tintR/G/B`.

6. Remove the old API methods: `setGlassRect`, `setGlassParams`, `setGlassTint` (both declaration and implementation).
