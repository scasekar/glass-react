  <action>
Create the web scaffold from scratch in the project root (not inside engine/).

**package.json** must include:
- Dependencies: react, react-dom, vite (as devDep), @vitejs/plugin-react (devDep), vite-plugin-wasm (devDep), vite-plugin-top-level-await (devDep), vite-plugin-live-reload (devDep), typescript (devDep), @types/react (devDep), @types/react-dom (devDep), nodemon (devDep), concurrently (devDep)
- Scripts:
  - "build:wasm": "cd engine && emcmake cmake -B build-web && cmake --build build-web -j4"
  - "watch:wasm": "nodemon --watch engine/src -e cpp,h --exec 'npm run build:wasm'"
  - "dev:vite": "vite"
  - "dev": "concurrently \"npm run watch:wasm\" \"npm run dev:vite\""
  - "build": "npm run build:wasm && vite build"
- "type": "module" for ESM support

**vite.config.ts** must:
- Import and use plugins: @vitejs/plugin-react, vite-plugin-wasm, vite-plugin-top-level-await
- Import vite-plugin-live-reload and configure it to watch: ['engine/build-web/**/*.{js,wasm}']
- Set optimizeDeps.exclude: ['engine'] to prevent Vite from pre-bundling WASM output
- Set server.headers for Cross-Origin-Opener-Policy: 'same-origin' and Cross-Origin-Embedder-Policy: 'require-corp' (needed for SharedArrayBuffer if threads are used later)
- Set server.fs.allow to include the project root and engine/build-web/ so Vite can serve the WASM files

**tsconfig.json** must:
- Target ES2022, module ESNext, moduleResolution bundler
- Include jsx: "react-jsx"
- Include src/ in the include array
- Set strict: true

**index.html** (at project root, Vite convention) must:
- Have a canvas element with id="gpu-canvas" and width="512" height="512"
- Have a div with id="root" for React mounting
- Load src/main.tsx as a module script
- Style: body with margin 0, dark background (#111), centered canvas
- IMPORTANT: The canvas MUST appear in the DOM before the WASM module initializes, because C++ references it by selector "#gpu-canvas"

**src/main.tsx** must:
- Import React and createRoot from react-dom/client
- Import App component
- Mount App to #root div

**src/App.tsx** must:
- Import and call the WASM loader on component mount (useEffect with empty deps)
- Display a status message while loading ("Initializing WebGPU engine...")
- Display error message if loading fails
- On success, display "Engine running" status text below the canvas
- Keep it minimal -- this is a proof-of-concept, not the final React architecture

**src/wasm/loader.ts** must:
- Load the Emscripten JS glue file (engine/build-web/engine.js)
- IMPORTANT: Emscripten's generated JS creates a Module factory function. The loader must:
  1. Import or load the engine.js glue file
  2. Call the Module factory (it returns a promise)
  3. The Module factory initializes WASM, runs main(), which calls Init() and starts the render loop
  4. Export a function like `initEngine()` that returns a Promise<void>
- Handle the case where the glue file loads the .wasm via relative path -- ensure the .wasm file is accessible from Vite's dev server
- APPROACH: Copy engine/build-web/engine.js and engine.wasm to public/ directory as part of build:wasm script, OR configure Vite to serve from engine/build-web/. The simpler approach is to add a postbuild copy step. Try loading the module via a dynamic import first; if that doesn't work with Emscripten's CommonJS-style output, fall back to loading via a script tag.
- PRACTICAL APPROACH: Since Emscripten's JS glue is not a standard ESM module, the most reliable method is:
  1. In the build:wasm script, copy engine.js and engine.wasm to public/wasm/
  2. In loader.ts, create a script element that loads /wasm/engine.js
  3. The Emscripten glue will self-initialize, find engine.wasm at /wasm/engine.wasm, and run main()
  4. OR: Use the Module factory pattern where Emscripten exports `createModule` when built with EXPORT_ES6=1 and MODULARIZE=1

**REVISED BUILD APPROACH:** Add to CMakeLists.txt link options: "-sMODULARIZE=1" and "-sEXPORT_ES6=1" and "-sEXPORT_NAME=createEngineModule". This makes the Emscripten output an ESM module with a factory function, which Vite can import natively. Then loader.ts simply does:
```typescript
import createEngineModule from '../../engine/build-web/engine.js';
const module = await createEngineModule();
```
If this approach fails during implementation (Vite can't resolve the Emscripten ESM output), fall back to the script-tag approach with public/ copy.

UPDATE engine/CMakeLists.txt to add these additional link options:
- "-sMODULARIZE=1"
- "-sEXPORT_ES6=1"
- "-sEXPORT_NAME=createEngineModule"

And rebuild the WASM to verify these options work with emdawnwebgpu.
