  <name>Task 1: Create C++ engine source and CMake build configuration</name>
  <files>engine/CMakeLists.txt, engine/src/main.cpp</files>
  <action>
Create the engine/ directory structure with CMakeLists.txt and src/main.cpp.

**engine/CMakeLists.txt** must:
- Set cmake_minimum_required(VERSION 3.22) and project(glass-engine LANGUAGES CXX)
- Set CMAKE_CXX_STANDARD 20
- Add executable target "engine" from src/main.cpp
- When EMSCRIPTEN is defined:
  - Set output suffix to ".js" (NOT ".html" -- we load the JS glue from Vite, not Emscripten's HTML shell)
  - Link emdawnwebgpu_cpp library via target_link_libraries
  - Add link options: "-sASYNCIFY=1", "-sALLOW_MEMORY_GROWTH", "-lembind"
  - Add "-sEXPORTED_RUNTIME_METHODS=['ccall']" for potential direct calls
  - Add "-sENVIRONMENT=web" to target web-only (skip Node.js checks in glue)
  - Add "--use-port=emdawnwebgpu" to the compile and link options (CRITICAL: this is how emdawnwebgpu is activated)
  - NOTE: Do NOT add --use-port to just link options; it must also be in compile options for the headers to be found
- Do NOT use -sUSE_WEBGPU (deprecated, unmaintained)
- Do NOT include ASYNCIFY_ONLY for now (accept larger binary per research recommendation)

**engine/src/main.cpp** must:
- Include webgpu/webgpu_cpp.h, iostream, and emscripten headers (emscripten.h, html5.h) under #ifdef __EMSCRIPTEN__
- Declare global wgpu::Instance, wgpu::Adapter, wgpu::Device, wgpu::Surface, wgpu::TextureFormat
- Implement Init() function:
  - Create instance with default descriptor: wgpu::CreateInstance() (no args -- Emscripten requires default)
  - Request adapter via instance.RequestAdapter() with CallbackMode::WaitAnyOnly + instance.WaitAny()
  - Request device via adapter.RequestDevice() with CallbackMode::WaitAnyOnly + instance.WaitAny()
  - Set uncaptured error callback on device descriptor for debugging
  - Print "WebGPU device acquired successfully" to cout on success
- Implement InitGraphics() function:
  - Create surface from canvas using wgpu::SurfaceSourceCanvasHTMLSelector with selector "#gpu-canvas"
  - Get surface capabilities from adapter
  - Configure surface with device, first available format, 512x512 initial size
- Implement Render() function:
  - Get current surface texture
  - Create render pass with clear color {0.0f, 0.5f, 0.8f, 1.0f} (distinctive blue -- easy to verify visually)
  - Submit command buffer
- Implement main():
  - Call Init(), then InitGraphics()
  - Under #ifdef __EMSCRIPTEN__: call emscripten_set_main_loop(Render, 0, false)
  - Do NOT use a while-loop on web (blocks browser event loop)

Follow the exact code patterns from the research file (Pattern 1, 2, 3) -- they are verified from Chrome DevRel examples and kainino0x's reference implementation. Use wgpu:: C++ namespace (not raw C wgpuXxx functions).
  </action>
