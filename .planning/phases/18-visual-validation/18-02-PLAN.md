<task type="auto">
  <name>Task 1: Conditionally tune regressed presets</name>
  <files>demo/controls/presets.ts</files>
  <action>
    Read the 18-01-SUMMARY.md to get the v3.0 baseline scores for light and dark modes.

    **Decision logic — run tuning only for modes that regressed:**
    - If light score > 0.40%: run `npm run tune -- --mode light`
    - If dark score > 0.85%: run `npm run tune -- --mode dark`
    - If both scores are within baseline: skip tuning, proceed directly to Task 2

    **If tuning is required for a mode:**

    Stop any dev server on port 5173 first:
    ```
    kill $(lsof -ti:5173) 2>/dev/null || true
    ```

    Run the tuner (this takes many minutes — do not interrupt):
    ```
    npm run tune -- --mode light    # if light regressed
    npm run tune -- --mode dark     # if dark regressed
    ```

    The tuner runs up to 23 cycles across 3 phases (interior → edges → distortion) and prints progress per cycle. When it completes, it writes:
    - `pipeline/output/tuning/best-params-light.json` (or dark)
    - `pipeline/output/tuning/tune-log-light.json` (or dark)

    **After each tune run, copy the best-params values into demo/controls/presets.ts:**

    Read `pipeline/output/tuning/best-params-[mode].json` to get all parameter values.

    In `demo/controls/presets.ts`, update the appropriate entry in PRESETS:
    - For light: update `PRESETS['Clear Light']` — replace all numeric/array fields with values from best-params-light.json. Keep `...DEFAULTS` spread, then override only the fields with tuned values.
    - For dark: update `PRESETS['Clear Dark']` — same pattern with best-params-dark.json.

    Preserve the tint field as a tuple: `[r, g, b]` not `{"0":r,"1":g,"2":b}`.

    **If no tuning was needed:** Note in summary that both presets already met baseline — no changes to presets.ts needed.
  </action>

  </verify>
  <done>
    If tuning ran: demo/controls/presets.ts updated with best-params values for regressed modes. Both tune-log JSONs exist.

  </done>
