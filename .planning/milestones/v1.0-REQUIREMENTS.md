# Requirements: LiquidGlass-React-WASM

**Defined:** 2026-02-10
**Core Value:** Glass components that look and feel like Apple's Liquid Glass -- refraction of a dynamic background through UI elements must be visually convincing at 60FPS.

## v1 Requirements — ALL SHIPPED

### Build Pipeline

- [x] **BUILD-01**: C++ engine compiles to WASM via Emscripten with WebGPU support
- [x] **BUILD-02**: emdawnwebgpu bindings integrated and functional
- [x] **BUILD-03**: CMake build system with `emcmake` produces .wasm + .js glue
- [x] **BUILD-04**: Vite dev server loads and hot-reloads WASM module
- [x] **BUILD-05**: Library publishable to npm with .wasm assets included

### Background Engine

- [x] **ENGINE-01**: Procedural noise simulation runs in C++ at 60FPS
- [x] **ENGINE-02**: Full-canvas background rendering to WebGPU texture
- [x] **ENGINE-03**: Engine exposes `init()`, `update()`, `getTexture()`, `resize()` API
- [x] **ENGINE-04**: Canvas resize handled without crashes or distortion

### GPU Texture Bridge

- [x] **BRIDGE-01**: Single shared GPUDevice between C++ engine and React UI
- [x] **BRIDGE-02**: Zero-copy texture sharing -- C++ renders, JS samples same GPUTexture
- [x] **BRIDGE-03**: Proper GPU resource lifecycle (no leaks, clean teardown)
- [x] **BRIDGE-04**: Render pipeline synchronization at 60FPS

### Glass Visual Effects

- [x] **GLASS-01**: Background refraction/distortion via WGSL fragment shader
- [x] **GLASS-02**: Frosted glass blur with configurable intensity
- [x] **GLASS-03**: Configurable opacity/transparency per component
- [x] **GLASS-04**: Rounded corners via SDF or clip
- [x] **GLASS-05**: Anti-aliased edges on glass components
- [x] **GLASS-06**: Chromatic aberration effect with adjustable intensity
- [x] **GLASS-07**: Specular highlights from static light source
- [x] **GLASS-08**: Edge rim lighting effect
- [x] **GLASS-09**: Multiple refraction modes (standard, prominent)
- [x] **GLASS-10**: Morphing transitions between glass states

### React Components

- [x] **COMP-01**: GlassProvider context wrapper initializes WebGPU + WASM engine
- [x] **COMP-02**: GlassPanel component with glass refraction effect
- [x] **COMP-03**: GlassButton component with glass refraction effect
- [x] **COMP-04**: GlassCard component with glass refraction effect
- [x] **COMP-05**: Components accept style props (blur, opacity, cornerRadius, tint)

### Accessibility

- [x] **A11Y-01**: Respects prefers-reduced-transparency (falls back to opaque)
- [x] **A11Y-02**: Respects prefers-reduced-motion (disables animated background)
- [x] **A11Y-03**: Text on glass meets WCAG 2.1 AA contrast ratio (4.5:1)
- [x] **A11Y-04**: Dark/light mode adaptation via CSS media queries

### Demo & Documentation

- [x] **DEMO-01**: Full demo page showcasing all glass components
- [x] **DEMO-02**: Demo includes interactive controls for glass parameters
- [x] **DEMO-03**: API documentation for all public components and props

## v2 Requirements

### Interaction

- **INTER-01**: Pointer/mouse-driven specular response
- **INTER-02**: Gyroscope/device tilt interaction
- **INTER-03**: 3D tilt on hover effect

### Advanced Features

- **ADV-01**: Nested glass (child components sample parent glass output)
- **ADV-02**: Content-blur mode (frosted glass over DOM content)
- **ADV-03**: Multiple simulation modes (noise, fluid, custom)
- **ADV-04**: Video refraction

## Out of Scope

| Feature | Reason |
|---------|--------|
| WebGL fallback | WebGPU-only is the value proposition; doubles shader work |
| Server-side rendering | WebGPU is client-only |
| CSS-only glassmorphism mode | Defeats purpose of GPU pipeline |
| Non-WebGPU browser support | Target modern browsers only |
| Mobile gyroscope | Deferred to v2 |
