**Add image loading helper function (outside the component, module-level):**
```typescript
async function loadAndUploadWallpaper(module: EngineModule): Promise<void> {
  const response = await fetch(wallpaperUrl);
  if (!response.ok) {
    console.error('GlassProvider: failed to load wallpaper:', response.status);
    return;
  }
  const blob = await response.blob();

  // colorSpaceConversion: 'none' preserves raw sRGB values from JPEG
  // This prevents the browser from applying unwanted ICC profile conversion
  const bitmap = await createImageBitmap(blob, { colorSpaceConversion: 'none' });

  // Extract raw RGBA pixels via OffscreenCanvas
  const canvas = new OffscreenCanvas(bitmap.width, bitmap.height);
  const ctx = canvas.getContext('2d')!;
  ctx.drawImage(bitmap, 0, 0);
  const imageData = ctx.getImageData(0, 0, bitmap.width, bitmap.height);
  bitmap.close();

  const rgba = new Uint8Array(imageData.data.buffer);

  // Transfer to WASM heap and upload
  const ptr = module._malloc(rgba.byteLength);
  module.HEAPU8.set(rgba, ptr);
  module.uploadImageData(ptr, imageData.width, imageData.height);
  module._free(ptr);
}
```

**CRITICAL: Use `imageData.width` and `imageData.height` (not `bitmap.width`/`bitmap.height`) when calling `uploadImageData`, because `bitmap` has been `.close()`d at that point. Alternatively, capture width/height before close.**

Corrected version -- capture dimensions before closing bitmap:
```typescript
const width = bitmap.width;
const height = bitmap.height;
bitmap.close();
// ... use width, height ...
module.uploadImageData(ptr, width, height);
```

**Add mode sync useEffect (after the image loading effect):**
```typescript
// Sync backgroundMode prop to C++ engine
useEffect(() => {
  if (!ready || !moduleRef.current) return;
  // 0 = Image, 1 = Noise (matches C++ BackgroundMode enum)
  moduleRef.current.setBackgroundMode(backgroundMode === 'image' ? 0 : 1);
}, [backgroundMode, ready]);
```

The mode sync effect runs independently of image loading. If user sets `backgroundMode="image"` but the wallpaper hasn't loaded yet, the C++ engine's `render()` method already falls back to noise (it checks `hasImageTexture_` before using the image pipeline). Once the wallpaper finishes loading, it will automatically appear because the mode is already set.
