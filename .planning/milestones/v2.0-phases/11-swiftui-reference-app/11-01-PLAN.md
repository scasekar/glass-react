    Create a new Xcode project structure manually (file-by-file, no `xcodebuild` generation needed -- write the .pbxproj and Swift files directly).

    **Project configuration:**
    - Bundle ID: `com.glassreference.app` (or similar)
    - Deployment target: iOS 26.0
    - Swift language version: 6
    - iPhone only (no iPad)
    - Device family: iPhone 16 Pro Simulator target

    **Wallpaper asset:**
    - Copy `src/assets/wallpaper.jpg` to `ios-reference/GlassReference/Assets.xcassets/wallpaper.imageset/wallpaper.jpg`
    - Create proper `Contents.json` for the imageset (universal scale, original image)
    - This is the EXACT same wallpaper used in the web demo -- critical for pixel comparison

    **GlassReferenceApp.swift:**
    - `@main` entry point
    - `@State private var colorScheme: ColorScheme = .light`
    - Pass `$colorScheme` binding to ContentView
    - Apply `.preferredColorScheme(colorScheme)` on the WindowGroup content

    **ContentView.swift — single-screen layout with ZStack:**

    Layer 1 — Full-bleed wallpaper:
    ```swift
    Image("wallpaper")
        .resizable()
        .aspectRatio(contentMode: .fill)
        .ignoresSafeArea()
    ```

    Layer 2 — Glass UI elements in VStack:
    All four canonical element types per user decision:

    1. **Navigation bar** (top):
       - HStack with chevron.left icon, "Reference" title, ellipsis icon
       - `.padding(.horizontal, 16).padding(.vertical, 12)`
       - `.glassEffect(glassVariant, in: RoundedRectangle(cornerRadius: 16))`

    2. **Search bar** (capsule/rounded):
       - HStack with magnifyingglass icon, "Search" placeholder text
       - `.padding(.horizontal, 16).padding(.vertical, 10)`
       - `.glassEffect(glassVariant, in: .capsule)`

    3. **Large glass panel** (card/sheet):
       - VStack with "Glass Panel" title, descriptive body text
       - `.frame(maxWidth: .infinity, alignment: .leading).padding(20)`
       - `.glassEffect(glassVariant, in: RoundedRectangle(cornerRadius: 28))`

    4. **Floating pill button** (bottom):
       - "Action Button" text with horizontal/vertical padding
       - `.glassEffect(glassVariant, in: .capsule)`

    Layer 3 — Controls overlay (hidden in capture mode):
    - `@State private var glassVariant: Glass = .regular`
    - `@State private var captureMode: Bool = false`
    - Three buttons in an HStack at bottom:
      - Toggle variant: shows "Regular" or "Clear", switches between `.regular` and `.clear`
      - Toggle mode: shows "Light" or "Dark", toggles `colorScheme` binding
      - Toggle capture mode: shows "Capture" or "Show UI", hides/shows controls overlay
    - Controls styled with `.background(.ultraThinMaterial).cornerRadius(12)`
    - Controls positioned with `Spacer()` pushing to bottom, `.padding(.bottom, 60)` for safe area

    **CRITICAL constraints (locked decisions):**
    - Use `.glassEffect(.regular)` and `.glassEffect(.clear)` ONLY -- zero customization, no `.tint()`, no `.interactive()`, no custom materials
    - The `glassVariant` state variable is of type `Glass` (Swift enum from SwiftUI framework)
    - Static glass only -- no scroll effects, no dynamic content
    - Wallpaper uses `.scaledToFill()` + `.ignoresSafeArea()` matching web demo's CSS `object-fit: cover`
    - Glass elements should respect safe areas naturally (only wallpaper ignores safe area)

    **Design for future wallpaper extensibility:**
    - Use a `@State private var wallpaperName: String = "wallpaper"` and `Image(wallpaperName)` so adding wallpapers later only requires adding assets and a picker (but do NOT build the picker now -- deferred)
